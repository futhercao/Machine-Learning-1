# 赛道一：ModelNet40 三维点云分类 — 设计思路

## 1. 任务回顾
- 输入：每个样本 10000 个点，每点 6 维 `(x, y, z, nx, ny, nz)`。
- 输出：40 类形状标签。
- 评分目标：实例准确率 ≥ 92%、类平均准确率 ≥ 90%。

## 2. 数据预处理
`preprocess.py` 把 `modelnet40_train.txt` 中的样本逐文件读入，统一为 `(N=9843, P=10000, 6)` 的 `float16` 张量保存为 `preprocessed/data.npy`，同时落地 `labels.npy / ids.npy / shape_names.npy`。
- 用 `fp16` 把整张表压到 ~1.2 GB，可直接 `mmap` 读，避免训练时大量小文件 IO。
- 标签从形状名中提取（`sample_id` 去掉末尾下划线数字段）。

## 3. 数据加载与增强（`data_loader.py`）
- **分层划分**：`stratified_split` 按类别 10% 留出做验证，保证每类都覆盖。
- **每个 epoch 随机采样 1024 个点**（点云分类的标准做法，等价于一种隐式增强）。
- 训练时的增强（仅作用于 xyz，法线随旋转矩阵同步变换）：
  - 绕 Z 轴随机旋转
  - 各向异性缩放 0.8–1.25
  - 平移 ±0.1
  - 微抖动 N(0, 0.01²) 并截断到 ±0.05
  - 点 dropout（随机把 ≤12.5% 的点复制到其它索引，等价随机丢点）
- 验证 / 推理：固定不增强。

## 4. 模型设计（`models.py`）
为了保证集成的多样性，准备了三类基线，全部支持 6 通道（xyz + 法线）输入：

| 名称 | 说明 | 参数量 |
| --- | --- | --- |
| `pointnet2_ssg` | PointNet++ Single-Scale Grouping：SA(512, r=0.2, k=32) → SA(128, r=0.4, k=64) → SA_all → FC | 1.48M |
| `pointnet2_msg` | Multi-Scale Grouping：两层 SA 各拼三个 (r, k) 尺度的局部特征 | 1.7M |
| `dgcnn` | 基于 k-NN(20) 的 EdgeConv 堆叠 + 1024 维全局嵌入 | 1.8M |

实现要点：
- FPS（最远点采样）+ ball query 的 grouping 用纯 PyTorch 实现，无外部 CUDA 算子，部署门槛低。
- DGCNN 的 knn 在 GPU 上用矩阵差实现，batch 内并行。

## 5. 训练（`train.py`）
- AdamW + warmup-cosine 学习率（warmup 占 2%，min ratio = 0.001），label smoothing = 0.1，gradient clip = 1.0。
- 默认 200 epoch、batch_size 32（DGCNN 因显存需求改 24），早停 patience = 40。
- 每个 epoch 同时记录 instance acc 与 class acc，按 instance acc 选择 best；保存 `args / val_iacc / val_cacc / epoch` 与 `state_dict`。

## 6. 多模型 + TTA 集成（`predict.py`）
- 同时加载若干 ckpt（`--ckpts a.pt b.pt ...`），文件名 `<model>_s<seed>.pt` 用来反推模型名。
- TTA：对每个样本进行 `--tta` 次随机抽取 1024 个点的子采样，每次推理后 softmax 求平均，再跨模型求平均。
- 输出 `id,class_name` 的 CSV（无表头），class_name 来自训练时保存的 `shape_names.npy`。
